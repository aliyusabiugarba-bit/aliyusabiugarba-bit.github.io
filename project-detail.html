The Daily Grind

A coffee-shop order ticket that doubles as a home-barista survey form.

The brief

Build a survey form covering a handful of standard input types — text, email, number, a dropdown, radio buttons, checkboxes, and a textarea — that still passes HTML5 validation. The catch: don't make it look like a generic form.

The approach

Instead of a plain feedback form, I built it as a "Home Barista Survey" styled like a coffee shop order ticket. The theme gave every field an actual reason to exist — cups per day as the number input, brew method as the radio group, gear owned as checkboxes — instead of arbitrary placeholder questions.

The visual details followed the same logic: a perforated tear-line at the top and bottom of the ticket, a rotated rubber-stamp badge, and a monospace receipt-style type pairing, so the page reads like an actual object rather than a stretched-out form.

Highlights

  • HTML5 validation on the email and number fields, including a min/max range on the number input.
  • A CSS-only perforated edge effect using a repeating radial gradient, no images involved.
  • Custom accent-color on radio buttons and checkboxes to match the ticket's accent without extra markup.
